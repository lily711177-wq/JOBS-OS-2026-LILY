### Layer 8 — Document Generation (The Output Layer)

Every document leaves the system in one of two states: **git (placeholder)** or **local (real-name)**. The split is non-negotiable. Git never sees your real phone, email, or name. Local files always carry your real identity. The generator decides format based on what the company's ATS eats best — DOCX for Workday/Greenhouse/Lever, PDF for Rippling and explicit PDF-preference companies. Never both. Never the wrong one.

**Format selection rules:**
| ATS Platform | Format | Why |
|---|---|---|
| Workday, Taleo, SuccessFactors, SAP, Oracle | DOCX | XML-based parser — needs structured DOCX |
| Greenhouse, Lever, Ashby, Gem | DOCX | Text-extraction parsers — DOCX preferred |
| SmartRecruiters, iCIMS | DOCX | Industry standard |
| Amazon internal | DOCX | Direct upload, no ATS intermediary |
| Rippling | PDF | Explicit requirement |
| Any ATS with "PDF preferred" in spec | PDF | User's intentional override |
| Any ATS with "DOCX preferred" in spec | DOCX | User's intentional override |
| Custom / unknown | DOCX | Safest default (most ATS parse DOCX reliably) |

**Document quality standards (every file, every time):**
- **Name:** 16pt bold, centered — the only element larger than body
- **Contact:** 9pt, centered — phone, email, LinkedIn in one line
- **Section headers:** 11pt bold with thin bottom border — no extra decoration
- **Body text:** 10pt Calibri, regular weight — consistent through all paragraphs
- **Bullets:** 0.30in left indent, 10pt — no hanging orphans
- **Margins:** 0.7in top/bottom, 0.8in left/right (resume); 1.0in all sides (cover)
- **Cover letters:** Greeting → body paragraphs with spacing → closing signature
- **No tables, no graphics, no columns, no headers/footers with content** — ATS must parse cleanly
- **Selectable text always** — never an image-based PDF

**Generation sequence (run AFTER every SHOOT approval or whenever files need refresh):**
```
1. Read APPLICATION_PACKAGE.md → extract resume text + cover letter text
2. Replace placeholders with real details ([NAME], real phone/email/LinkedIn)  
3. Parse ATS line → select DOCX or PDF (never both)
4. Apply professional formatting per standards above
5. Write to [Company]/OUTPUT/ (excluded from git via .gitignore)
6. Verify: open file, confirm alignment, spacing, name, contact
```

**Three-way sync rule:** GitHub holds canonical placeholders. E: drive holds local real-name files. Current machine holds working copy. These three must always be identical in structure — only the OUTPUT/ directory differs between machines. The SYNC command runs FIRST on every new machine to establish this invariant.
