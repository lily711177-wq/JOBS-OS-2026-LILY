# HARDWARE & WORK-TYPE MAPPING
## Core Kernel Module | Know Your Device Before Day 1
## Shapeshifter Edition — Agnostic, Works for Any Company

---

### PRINCIPLE

> Before you accept an offer, you should already know what laptop
> you'll open on Day 1, what OS you'll run, what communication tools
> they use, and how their IT works. This eliminates the "new job
> disorientation" period and lets you hit the ground running.

This module maps every company type → their standard hardware,
device issuance policy, IT environment, and work-style norms.
Use this to prep your muscle memory before Day 1.

### DEVICE PREP — Before Day 1

#### If they give you a Mac:

| Skill | Why | Pre-Learn |
|-------|-----|-----------|
| Cmd+Space (Spotlight) | App launcher, calculator, converter | Use it 50x/day |
| Cmd+Tab / Cmd+` | App switcher and window switcher | Muscle memory in 2 days |
| Trackpad gestures | 3-finger swipe, pinch, spread | Learn before Day 1 |
| macOS keyboard shortcuts | Cmd+C/V/X, Cmd+Q, Cmd+W | Tooklit for speed |
| Terminal | If they're dev-adjacent, you'll need it | Basic commands |
| Homebrew | macOS package manager | `brew install` basics |

#### If they give you a ThinkPad:

| Skill | Why | Pre-Learn |
|-------|-----|-----------|
| Windows key shortcuts | Win+1..9, Win+Tab, Win+D | Speed navigation |
| ThinkPad TrackPoint | The red nub — learn it or hate it | Give it 3 days |
| BitLocker recovery | You will need this at some point | Know where recovery key is |
| Corporate VPN | Always-on, always-connected | Get certs installed early |
| Outlook rules | You'll drown in email without them | Set up Day 1 |

#### Universal Prep (Any Device):

| Tool | Learn Before Day 1 | Why |
|------|-------------------|-----|
| **Slack** | Keyboard shortcuts, channels vs DMs, huddle, threads | You will live in Slack |
| **Notion** | Blocks, databases, linked pages, templates | Most companies use it |
| **Jira/Linear** | Issue types, workflows, boards | You'll track work here |
| **Google Docs** | Comment mode, suggest mode, version history | Docs are collaborative |
| **Office 365** | PPT shortcuts, Excel power-user tricks | Consulting lives in PPT |
| **Salesforce** | Basic navigation, reports, dashboards | RevOps lives in Salesforce |

### HOW TO USE THIS MODULE

1. **During application phase** — check the device map to know what
   computer you'll use. Prep mentally.
2. **During interview** — if relevant, reference the tooling:
   "I'm comfortable on Mac or Windows, whichever the team uses."
3. **After offer** — ask about hardware during offer negotiation.
   "What's the standard laptop?" shows you're thinking ahead.
4. **Before Day 1** — practice the keyboard shortcuts and tools
   listed above for the device you'll get.
5. **Day 1** — you already know what you're working with. Zero
   disorientation.

### INTEGRATION WITH INFILTRATION LAYER

When you know what device they use, you can reference it naturally:

**In interview:**
"I've been primarily on Mac the last few years, but I'm equally
comfortable on Windows — whatever the team uses."

**On Day 1 with a ThinkPad user at a Mac company:**
"First ThinkPad in a while — nice keyboard. I used one at my
last company before we switched to Macs."

**On Day 1 with a Mac at a PC company:**
"Glad you guys are on Mac — I've been on one for years.
Makes the transition seamless."

This is subtle infiltration — you're already familiar with
their environment.
